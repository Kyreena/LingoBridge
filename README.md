\# LingoBridge



LingoBridge is a web-based speech-to-sign language translation system designed to support inclusive classroom communication for learners with hearing impairments.



\## Project Overview

LingoBridge captures spoken English in real time, converts it into text using automated speech recognition, and maps recognized words or short phrases to a curated library of pre-recorded American Sign Language (ASL) video signs. The system displays ASL videos in near real time, enabling deaf and hard-of-hearing learners to follow classroom instruction as it happens.



This project is developed as a final-year undergraduate project for the Bachelor of Business Information Technology at Strathmore University.



\## Objectives

\- Support inclusive classroom communication for deaf and hard-of-hearing learners

\- Translate spoken English into ASL visual representations in real time

\- Provide a low-cost, browser-based alternative to human interpreters

\- Demonstrate the use of AI-driven speech recognition in education



\## Scope

\- Speech-to-text using browser APIs or Google Speech-to-Text

\- Text-to-ASL mapping using pre-recorded ASL videos

\- Web-based interface (no special hardware required)



\### Out of Scope (Phase 1)

\- ASL-to-speech translation

\- Avatar-based signing

\- Multilingual or non-ASL support



\## Technology Stack (Planned)

\- Frontend: HTML, CSS, JavaScript

\- Backend: Node.js or Flask

\- Speech Recognition: Web Speech API / Google Speech-to-Text

\- Data Storage: ASL video repository (file-based or database)



\## Methodology

The system follows a Modified Waterfall methodology supported by Structured Systems Analysis and Design (SSAD), progressing through:

1\. Requirements analysis

2\. System design (DFDs, ERDs)

3\. Implementation

4\. Testing and evaluation


## Version Control

from being committed to the repository.
